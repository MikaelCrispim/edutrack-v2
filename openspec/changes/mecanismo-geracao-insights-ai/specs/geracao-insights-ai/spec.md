## ADDED Requirements

### Requirement: Trigger AI Insight Generation
The system SHALL provide an endpoint to trigger the AI insight generation process.

#### Scenario: Frontend triggers insight generation
- **WHEN** the frontend sends a `POST` request to `/api/ai/generate-insights`
- **THEN** the backend SHALL initiate the insight generation process.

### Requirement: Rate Limit Insight Generation
The system SHALL prevent the insight generation process from running too frequently for a single user.

#### Scenario: User attempts to generate insights too soon
- **WHEN** a `POST` request is sent to `/api/ai/generate-insights` for a user who has had insights generated within the last 24 hours
- **THEN** the system SHALL return a response indicating that the request is rate-limited and not generate new insights.

#### Scenario: User generates insights after the rate limit period
- **WHEN** a `POST` request is sent to `/api/ai/generate-insights` for a user who has not had insights generated in the last 24 hours
- **THEN** the system SHALL proceed with the insight generation process.

### Requirement: Store Generated Insights
The system SHALL store the insights generated by the AI service in the `ai_insights` database table.

#### Scenario: Successful insight generation and storage
- **WHEN** the AI service successfully returns a list of insights
- **THEN** the system SHALL create a new record in the `ai_insights` table for each insight, linking it to the correct user.
- **AND** the system SHALL update the `last_insight_generated_at` timestamp for the user.
